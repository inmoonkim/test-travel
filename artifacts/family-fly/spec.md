# FamilyFly — spec.md

## 개요
가족 여행자가 출발지·목적지·날짜 범위·인원을 입력하면 항공+호텔 조합의 최저가를 날짜별 달력으로 탐색하고, 목표 가격 도달 시 이메일로 알림을 받는다.

---

## 범위

### 포함
- 출발지 / 목적지 / 출발일 범위 / 귀환일 범위 / 성인 수 / 소아 수 입력 폼
- 출발일 범위를 달력으로 표시하고 각 날짜 셀에 항공+호텔 합산 최저가(원화) 표시
- 날짜 셀 클릭 시 해당 날짜 출발 기준 항공+호텔 조합 리스트 표시
- 각 조합에서 항공사·호텔 외부 예약 페이지로 새 탭 이동
- 항공·호텔 각각 별도 목표가 설정 + 이메일 입력으로 알림 구독
- 가격이 목표가 이하로 하락 시 이메일 알림 자동 발송
- 이메일 내 링크로 알림 취소

### 제외
- 앱 내 결제·예약 — 항공사 GDS 연동 비용이 너무 크다. 외부 링크로 대체
- 회원가입·로그인 — 이메일 알림에 이메일 주소만 필요. 계정 관리 없음
- 가족 구성원 일정 공유 / 초대 — 날짜 탐색 핵심이 검증된 뒤 Phase 2
- 웹 푸시 알림 — 이메일 알림로 먼저 검증 후 결정
- 모바일 네이티브 앱
- 앱 내 직접 스크래핑 — SerpAPI 등 법적 책임이 위임된 중간 API 계층은 허용

## 시나리오

### 1. 검색 조건 입력 후 달력 뷰 표시

- **Given** — 메인 화면. 모든 필드(출발지, 목적지, 출발일 범위, 귀환일 범위, 성인 수)가 입력됨 (소아 0명 가능)
- **When** — "검색" 버튼 클릭
- **Then** — 로딩 인디케이터가 표시된다. 결과 수신 후 출발일 범위를 달력으로 렌더하고, 가격이 있는 날짜 셀에는 항공+호텔 합산 최저가(원화, 예: "₩320,000")가 표시된다

성공 기준:
- [ ] 유효한 조건 입력 → 검색 버튼 클릭 시 로딩 인디케이터가 나타난다
- [ ] 로딩 완료 후 출발일 범위(예: 6/10~6/20) 내 날짜들이 달력으로 표시된다
- [ ] 가격이 있는 날짜 셀에는 원화 금액("₩N")이 표시된다
- [ ] 가격 없는 날짜 셀에는 금액 대신 빈 상태 또는 "-" 가 표시된다

---

### 2. 날짜 선택 후 항공+호텔 조합 리스트 표시

- **Given** — 달력 뷰가 표시됨. 특정 날짜 셀에 가격이 표시됨
- **When** — 가격이 표시된 날짜 셀 클릭
- **Then** — 해당 출발일 기준 항공+호텔 조합 목록이 표시된다. 각 조합 카드에는 항공편명, 출발·도착 시각, 항공 가격, 호텔명, 호텔 1박 가격, 합산 가격이 나타난다

성공 기준:
- [ ] 날짜 셀 클릭 → 해당 날짜 출발 조합 리스트가 화면에 나타난다
- [ ] 각 조합 카드에 항공편명, 항공 가격(원화), 호텔명, 호텔 가격(원화), 합산 가격이 표시된다
- [ ] 조합 카드가 합산 가격 오름차순으로 정렬된다

---

### 3. 외부 예약 링크 이동

- **Given** — 항공+호텔 조합 리스트 표시됨
- **When** — 조합 카드의 "항공 예약" 링크 클릭
- **Then** — 해당 항공편을 예약할 수 있는 항공사 또는 제휴 사이트 페이지가 새 탭으로 열린다

성공 기준:
- [ ] "항공 예약" 클릭 → 새 탭에서 외부 항공 예약 페이지가 열린다
- [ ] "호텔 예약" 클릭 → 새 탭에서 외부 호텔 예약 페이지가 열린다
- [ ] 현재 탭의 앱 화면은 그대로 유지된다

---

### 4. 필수 입력 누락 시 오류 표시

- **Given** — 메인 화면. 목적지 필드가 비어 있음
- **When** — "검색" 버튼 클릭
- **Then** — 목적지 필드에 "목적지를 입력하세요" 오류 메시지가 표시되고 검색이 실행되지 않는다

성공 기준:
- [ ] 출발지 미입력 → "출발지를 입력하세요" 오류 표시, 검색 실행 안 됨
- [ ] 목적지 미입력 → "목적지를 입력하세요" 오류 표시
- [ ] 출발일 범위 미입력 → "출발일 범위를 선택하세요" 오류 표시
- [ ] 귀환일 범위 미입력 → "귀환일 범위를 선택하세요" 오류 표시
- [ ] 성인 수 0 → "성인은 최소 1명 이상이어야 합니다" 오류 표시

---

### 5. 귀환일 범위가 출발일 범위보다 이른 경우

- **Given** — 출발일 범위를 6/20~6/25로 입력한 상태
- **When** — 귀환일 범위를 6/10~6/15로 입력
- **Then** — "귀환일은 출발일보다 늦어야 합니다" 오류 메시지가 즉시(저장 전) 표시된다

성공 기준:
- [ ] 귀환일 범위 종료일이 출발일 범위 시작일보다 이른 경우 → 오류 메시지 표시
- [ ] 해당 오류 상태에서는 검색 버튼이 비활성화된다

---

### 6. 검색 결과 없음

- **Given** — 유효한 검색 조건 입력 완료
- **When** — "검색" 버튼 클릭, API가 해당 조건에 대한 결과를 반환하지 않음
- **Then** — 달력이 표시되지만 모든 날짜 셀이 비어 있고, "입력하신 조건에 맞는 항공+호텔 조합을 찾을 수 없습니다. 날짜 범위나 목적지를 조정해 보세요" 메시지가 표시된다

성공 기준:
- [ ] 결과 없음 → 빈 달력 + 안내 메시지 표시
- [ ] 빈 날짜 셀 클릭 → 아무 반응 없음 (리스트 진입 안 됨)

---

### 7. 가격 알림 설정

- **Given** — 달력 뷰 또는 조합 리스트 화면
- **When** — "알림 설정" 클릭 → 이메일 입력, 항공 목표가(원화) 입력, 호텔 목표가(원화) 입력 → "알림 저장" 클릭
- **Then** — "알림이 설정되었습니다. 입력하신 이메일로 가격이 목표가 이하로 떨어지면 알림을 보내 드립니다" 메시지가 표시된다

성공 기준:
- [ ] 유효한 이메일 + 항공 목표가 + 호텔 목표가 입력 → 알림 저장 성공 메시지 표시
- [ ] 항공 목표가만 입력하고 저장 가능 (호텔 목표가는 선택)
- [ ] 호텔 목표가만 입력하고 저장 가능 (항공 목표가는 선택)
- [ ] 항공·호텔 목표가 모두 미입력 → "항공 또는 호텔 목표 가격 중 하나 이상을 입력하세요" 오류 표시

---

### 8. 잘못된 이메일 형식으로 알림 저장 시도

- **Given** — 알림 설정 폼
- **When** — "abc" 등 이메일 형식이 아닌 값을 이메일 필드에 입력 후 "알림 저장" 클릭
- **Then** — "올바른 이메일 주소를 입력하세요" 오류 메시지가 이메일 필드 아래 표시된다. 알림이 저장되지 않는다

성공 기준:
- [ ] `abc`, `abc@`, `@domain.com` 등 → 이메일 오류 메시지 표시
- [ ] 유효한 이메일 형식(예: `user@example.com`) → 오류 없음

---

### 9. 가격 하락 시 이메일 알림 수신

- **Given** — 항공 목표가 10만원, 호텔 목표가 8만원으로 알림 설정됨. 이후 항공 최저가가 9만8천원으로 하락
- **When** — 시스템이 가격 하락을 감지
- **Then** — 설정된 이메일 주소로 알림 이메일이 발송된다. 이메일 본문에는 항공편명, 현재 가격(₩98,000), 이전 가격 대비 하락 금액, 예약 링크, 알림 취소 링크가 포함된다

성공 기준:
- [ ] 항공 가격이 항공 목표가 이하로 하락 → 이메일 발송됨 (제목에 "가격 하락 알림" 포함)
- [ ] 호텔 가격이 호텔 목표가 이하로 하락 → 이메일 발송됨
- [ ] 이메일 본문에 항공편명 또는 호텔명, 현재 가격, 이전 가격 대비 하락액이 표시됨
- [ ] 이메일 본문에 예약 가능한 외부 링크가 포함됨
- [ ] 이메일 본문에 "알림 취소" 링크가 포함됨

---

### 10. 이메일 링크로 알림 취소

- **Given** — 알림 이메일 수신됨
- **When** — 이메일 내 "알림 취소" 링크 클릭
- **Then** — 브라우저에서 "알림이 취소되었습니다" 확인 페이지가 표시된다. 이후 해당 이메일·조건으로 알림이 발송되지 않는다

성공 기준:
- [ ] 취소 링크 클릭 → "알림이 취소되었습니다" 페이지 표시
- [ ] 동일한 취소 링크를 다시 클릭 → "이미 취소된 알림입니다" 메시지 표시 (오류 아님)

---

## 불변 규칙

**외부 링크**: 항공사·호텔 예약 페이지는 항상 새 탭으로 열린다. 앱 자체 네비게이션으로 이동하지 않는다.

**통화**: 모든 가격은 원화(₩, KRW)로 표시한다.

**성인 최소 인원**: 성인 0명인 상태로는 검색이 실행되지 않는다. 폼 진입 시 기본값은 성인 2명, 소아 0명이다.
