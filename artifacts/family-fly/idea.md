# FamilyFly — 가족 여행 최저가 탐색기

## Problem Statement
HMW — 가족 여행자가 항공·호텔 사이트를 번갈아 오가며 날짜를 바꿔 비교하는 대신,
날짜 범위와 조건을 한 번 설정하면 지금 예약 가능한 최적 조합이 저절로 모이게 할 수 있을까?

## Recommended Direction
**날짜 범위 탐색 + 가격 알림** 조합.

가족 여행의 진짜 고통은 두 가지다: (1) 여러 날짜 조합을 직접 바꿔가며 비교해야 하고,
(2) 결정하는 사이 가격이 오른다. 날짜 범위를 입력하면 항공+호텔 조합 가격을 달력 뷰로
보여주고, 원하는 가격대에 도달했을 때 알림을 보낸다.

스카이스캐너는 항공 단일 알림을 제공하지만 호텔 조합 알림은 없다. 구글 항공편은 가격
캘린더가 있지만 역시 항공만 다룬다. 항공+호텔 통합 날짜 탐색은 아직 잘 된 서비스가 없다.

가족 구성원 일정 공유(C방향)는 이 핵심이 검증된 뒤 Phase 2.

## Key Assumptions to Validate
- [ ] **[Must]** Amadeus API(항공) + Booking.com Affiliate/RapidAPI(호텔)가 합리적 한도·비용에
     실시간 데이터를 제공한다 → 빌드 전 API 가입해서 rate limit·가격 확인
- [ ] **[Must]** 알림을 받은 사용자가 외부 예약 링크로 실제 예약 행동까지 이어진다
     → 웹 푸시 + 딥링크 포함 여부를 MVP에 결정
- [ ] **[Should]** 가족 여행자가 날짜 범위 슬라이더를 실제로 조작하며 최저가를 탐색한다
     → 첫 사용자 5명 관찰 테스트로 검증

## MVP Scope
**포함:**
- 출발지 / 목적지 / 날짜 범위(±7일) / 인원 입력
- 날짜별 항공+호텔 조합 최저가 달력 뷰
- 조합 클릭 시 외부 예약 링크(항공사, Booking.com)
- 목표 가격 설정 → 가격 도달 시 이메일 알림

**미포함:**
- 앱 내 직접 결제
- 가족 구성원 일정 공유/초대
- 모바일 네이티브 앱

## Not Doing (and Why)
- **직접 예약 연동** — 항공사 GDS 연동은 수개월·수천만원 수준. 핵심 가정 검증 전 투자 비합리적
- **가족 코디네이터(공유 기능)** — 가격 탐색 자체가 유용한지 먼저 검증 후 추가
- **스크래핑** — 법적 리스크 + 유지비 크고, Amadeus 무료 티어로 시작 가능
- **모바일 앱** — 웹 푸시 알림으로 먼저 검증. 채택률 확인 후 결정

## Open Questions

- 웹 푸시 알림만으로 재방문율을 만들 수 있는가, 아니면 이메일이 더 효과적인가?
